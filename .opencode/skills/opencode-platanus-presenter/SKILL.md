# Platanus Presenter Skill

Use this skill with the `presenter` agent.

## Procedure

1. Read the review report and build log.
2. Identify the actual working demo path and do not claim anything beyond it.
3. Write the one-liner, 30-second pitch, and 2-minute demo script.
4. Create Markdown slides around problem, solution, demo, how it works, why now, and close.
5. Prepare fallback scripts for deploy, API, Wi-Fi, and audio failure.
6. Prepare judge Q&A and public-vote copy.
7. Ensure sponsor usage is explained naturally.

## Output

Create or update `artifacts/05_demo_pack.md`.

## Guardrails

- Be honest.
- Keep it short.
- Make the user pain and wow moment obvious.
- Do not modify core product code unless explicitly asked.
